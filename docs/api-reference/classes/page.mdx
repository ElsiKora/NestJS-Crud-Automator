# Classes

Complete reference for all classes provided by NestJS CRUD Automator.

## Base Classes

### ApiServiceBase

Base class for services providing CRUD operations.

**Location:** `src/class/api/service-base.class.ts`

**Methods:**

#### `create(properties, eventManager?): Promise<E>`

Creates a new entity.

```typescript
async create(
  properties: TApiFunctionCreateProperties<E>,
  eventManager?: EntityManager
): Promise<E>
```

#### `get(properties, eventManager?): Promise<E>`

Retrieves a single entity.

```typescript
async get(
  properties: TApiFunctionGetProperties<E>,
  eventManager?: EntityManager
): Promise<E>
```

#### `getList(properties, eventManager?): Promise<IApiGetListResponseResult<E>>`

Retrieves a paginated list of entities.

```typescript
async getList(
  properties: TApiFunctionGetListProperties<E>,
  eventManager?: EntityManager
): Promise<IApiGetListResponseResult<E>>
```

#### `getMany(properties, eventManager?): Promise<E[]>`

Retrieves multiple entities.

```typescript
async getMany(
  properties: TApiFunctionGetManyProperties<E>,
  eventManager?: EntityManager
): Promise<E[]>
```

#### `update(criteria, properties, eventManager?): Promise<E>`

Updates an entity.

```typescript
async update(
  criteria: TApiFunctionUpdateCriteria<E>,
  properties: TApiFunctionUpdateProperties<E>,
  eventManager?: EntityManager
): Promise<E>
```

#### `delete(criteria, eventManager?): Promise<void>`

Deletes an entity.

```typescript
async delete(
  criteria: TApiFunctionDeleteCriteria<E>,
  eventManager?: EntityManager
): Promise<void>
```

## Utility Classes

### MetadataStorage

Singleton class for storing and retrieving entity property metadata.

**Location:** `src/class/metadata-storage.class.ts`

**Methods:**

#### `getInstance(): MetadataStorage`

Returns the singleton instance.

```typescript
const storage = MetadataStorage.getInstance();
```

#### `setMetadata(entityName, propertyKey, metadataKey, metadata)`

Stores metadata for a property.

```typescript
storage.setMetadata("UserEntity", "username", "api-property-describe", { ... });
```

#### `getMetadata(entityName, propertyKey, metadataKey)`

Retrieves metadata for a property.

```typescript
const metadata = storage.getMetadata("UserEntity", "username", "api-property-describe");
```

#### `getAllMetadata(entityName)`

Retrieves all metadata for an entity.

```typescript
const allMetadata = storage.getAllMetadata("UserEntity");
```

#### `hasMetadata(entityName, propertyKey, metadataKey): boolean`

Checks if metadata exists.

```typescript
const exists = storage.hasMetadata("UserEntity", "username", "api-property-describe");
```

- [Core Concepts - Metadata Storage](/docs/nestjs-crud-automator/core-concepts/metadata-storage)

## Authorization Classes

### ApiAuthorizationPolicyBase

Base class for hooks-mode authorization policies. Provides helper methods such as `allow`, `deny`, `allowForRoles`, `allowForPermissions`, `denyForPermissions`, and `scopeToOwner` for constructing rules executed by the hooks engine.

**Location:** `src/class/api/authorization/policy/base.class.ts`

### ApiAuthorizationModule

Global module that wires the authorization registry, discovery service, guard, hooks runtime, IAM runtime, bootstrap validation, and simulator services.

**Location:** `src/module/api/authorization.module.ts`

**Exports:** `ApiAuthorizationPolicyDiscoveryService`, `ApiAuthorizationRuntime`, `ApiAuthorizationSimulator`, `ApiAuthorizationBootstrapValidationService`, `AUTHORIZATION_POLICY_REGISTRY_TOKEN`

**Static methods:**

- `forRoot(options?: IApiAuthorizationModuleOptions): DynamicModule` for value-based resolver and source registration
- `forRootAsync(options: IApiAuthorizationModuleAsyncOptions): DynamicModule` for DI-backed `principalResolver`, `hookPermissionSources`, `iam.attachmentSources`, and `iam.documentSources`

`forRootAsync(...)` supports the standard Nest async module patterns: `imports`, `inject`, `useFactory`, `useClass`, and `useExisting`. Use `IApiAuthorizationModuleOptionsFactory` when you want a dedicated Nest provider class that builds the authorization module options.

### ApiAuthorizationRuntime

Shared runtime that resolves the principal, selects the controller or route authorization mode, and dispatches to either hooks or IAM evaluation.

**Location:** `src/class/api/authorization/runtime.class.ts`

### ApiAuthorizationSimulator

Service for evaluating authorization decisions outside HTTP requests by reusing the same runtime as the guard.

**Location:** `src/class/api/authorization/simulator.class.ts`

### ApiAuthorizationIamEngine

Evaluates IAM attachments and policy documents, applies explicit deny and boundary rules, resolves runtime condition paths such as `principal.*`, `resource.*`, and `request.*`, and produces IAM decision traces.

**Location:** `src/class/api/authorization/iam/engine.class.ts`

### ApiAuthorizationIamQueryPlanner

Compiles safe `resource.*` IAM conditions into database scopes for controller queries. The planner supports `StringEquals` and `StringEqualsIfExists` on filterable resource fields, including `resource.id`, `resource.operatorId`, and nested `queryPath` mappings such as `operator.id`.

**Location:** `src/class/api/authorization/iam/query-planner.class.ts`

### ApiAuthorizationHookPermissionResolver

Resolves and caches permissions for hooks mode from configured `IApiHookPermissionSource` implementations.

**Location:** `src/class/api/authorization/hook-permission-resolver.class.ts`

### ApiAuthorizationBootstrapValidationService

Validates securable controller configuration, mode selection, hooks policy presence, and IAM prerequisites during application bootstrap.

**Location:** `src/class/api/authorization/bootstrap-validation.service.class.ts`
