# `@ApiControllerSecurable`

Marker-only decorator that enables the authorization pipeline for a controller. When omitted, `ApiAuthorizationGuard` short-circuits and no hooks or IAM evaluation runs.

## Signature

```typescript
@ApiControllerSecurable()
```

## Behavior

- Sets `CONTROLLER_API_DECORATOR_CONSTANT.SECURABLE_METADATA_KEY` to `true`.
- Does not choose `hooks` vs `iam`.
- Does not declare `policyNamespace`, `resourceDefinition`, sources, or fallback behavior.
- Must be paired with `@ApiController({ authorization: ... })` on any secured controller.
- Requires custom `@ApiMethod(...)` handlers on the controller to declare `authorization.action`.
- Is validated at bootstrap: missing `authorization` on a securable controller is treated as a configuration error.

## Example

```typescript
@Controller("users")
@ApiControllerSecurable()
@ApiController<UserEntity>({
	authorization: {
		defaultMode: EApiAuthorizationMode.HOOKS,
	},
	entity: UserEntity,
	path: "users",
})
export class UserController {}
```

## Notes

- Keep this decorator even in IAM mode.
- Use route-level `authorization.mode` inside `@ApiController(...)` when a specific route must differ from the controller default.
- Each route still runs in exactly one authorization mode.
- Auto-generated CRUD routes receive their authorization action strings automatically; bespoke `@ApiMethod(...)` routes must provide one explicitly.
