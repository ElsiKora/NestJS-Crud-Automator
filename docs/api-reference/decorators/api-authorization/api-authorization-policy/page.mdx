# `@ApiAuthorizationPolicy`

Registers a class as an authorization policy for a specific entity. Policies execute before every secured CRUD route and can allow/deny requests based on custom logic.

## Signature

```typescript
@ApiAuthorizationPolicy<E extends IApiBaseEntity>({
	entity,
	priority?,
	policyId?,
	description?,
}: IApiAuthorizationPolicySubscriberProperties<E>)
```

## Options

| Option        | Description                                                                                                          |
| ------------- | -------------------------------------------------------------------------------------------------------------------- |
| `cache`       | Optional cache override for this policy (`isEnabled`, `ttlMs`).                                                      |
| `entity`      | Entity this policy protects. Required.                                                                               |
| `priority`    | Higher priorities execute earlier. Defaults to `0`.                                                                  |
| `policyId`    | Optional custom identifier. Defaults to `<entityName>Policy`. Useful when multiple policies protect the same entity. |
| `description` | Short summary displayed in tooling or logs.                                                                          |

## Usage steps

1. Extend `ApiAuthorizationPolicyBase<E>` and implement lifecycle hooks (`onBeforeCreate`, `onBeforeGetList`, etc.).
2. Decorate the class with `@ApiAuthorizationPolicy`.
3. Import `ApiAuthorizationModule` so policies are discovered. Use `ApiAuthorizationModule.forRoot({ subjectResolver })` when you need a custom subject resolver.
4. Add `@ApiControllerSecurable()` (and optionally `@ApiController`) to controllers that should enforce policies.

## Example

```typescript
import type {
	IApiAuthorizationRuleContext,
	IApiAuthorizationScope,
	TApiAuthorizationPolicyBeforeCreateContext,
	TApiAuthorizationPolicyBeforeGetListContext,
	TApiAuthorizationPolicyBeforeUpdateContext,
} from "@elsikora/nestjs-crud-automator";

import { ApiAuthorizationPolicy, ApiAuthorizationPolicyBase, EAuthorizationPermissionMatch } from "@elsikora/nestjs-crud-automator";

@Injectable()
@ApiAuthorizationPolicy<UserEntity>({
	entity: UserEntity,
	priority: 150,
	description: "Allow platform admins and operator admins to manage users",
})
export class UserAccessPolicy extends ApiAuthorizationPolicyBase<UserEntity> {
	private scopeToOperator(context: IApiAuthorizationRuleContext<UserEntity>): IApiAuthorizationScope<UserEntity> {
		return {
			where: {
				operatorId: context.subject.attributes?.operatorId,
			},
		};
	}

	public onBeforeCreate(context: TApiAuthorizationPolicyBeforeCreateContext<UserEntity>) {
		if (context.body.operatorId !== undefined && context.body.operatorId !== context.subject.attributes?.operatorId) {
			return [];
		}

		return [
			...this.allowForRoles(["platform-admin"], {
				description: "Platform admins can create users anywhere",
			}),
			...this.allowForPermissions(["admin.user.create"], {
				description: "Operator admins can create users only in their operator",
				scope: (ruleContext: IApiAuthorizationRuleContext<UserEntity>) => this.scopeToOperator(ruleContext),
			}),
		];
	}

	public onBeforeGetList(context: TApiAuthorizationPolicyBeforeGetListContext<UserEntity>) {
		if (context.query.filters?.operatorId && context.query.filters.operatorId !== context.subject.attributes?.operatorId) {
			return [];
		}

		return [
			...this.allowForRoles(["platform-admin"], {
				description: "Platform admins can list users without scope restrictions",
			}),
			...this.allowForPermissions(["admin.user.read", "admin.user.list"], {
				description: "Operator admins need both read and list permissions",
				match: EAuthorizationPermissionMatch.ALL,
				scope: (context: IApiAuthorizationRuleContext<UserEntity>) => this.scopeToOperator(context),
			}),
		];
	}

	public onBeforeUpdate(context: TApiAuthorizationPolicyBeforeUpdateContext<UserEntity>) {
		const wantsPlatformAdmin = context.body.role === "platform-admin";

		return [
			...(wantsPlatformAdmin
				? this.deny({
						description: "Operator admins may never assign platform-admin through update payloads",
						priority: 1500,
					})
				: []),
			...this.allowForPermissions(["admin.user.update"], {
				description: "Operator admins can update users inside their operator",
				scope: (context: IApiAuthorizationRuleContext<UserEntity>) => this.scopeToOperator(context),
			}),
			...this.denyForPermissions(["admin.user.update"], {
				condition: ({ subject }: IApiAuthorizationRuleContext<UserEntity>) => Boolean(subject.attributes?.isOperatorLocked),
				description: "Locked operator admins are denied even if an allow also matches",
				priority: 1000,
			}),
		];
	}
}
```

### Helper methods

`ApiAuthorizationPolicyBase` exposes helpers for the most common rule shapes:

- `allow()` / `deny()` for fully custom rules
- `allowForRoles()` for role-based checks
- `allowForPermissions()` for permission-based checks
- `denyForPermissions()` for explicit permission-based denies
- `scopeToOwner()` for owner scoping

### Hook context

Policy hooks receive:

- `subject`, `authenticationRequest`, `entity`, `entityMetadata`, `action`, and `routeType`
- request metadata via `body`, `parameters`, `query`, `headers`, and `ip`
- the same values again under `context.DATA` for parity with the rest of the subscriber system

For stronger route-aware typing, use:

- `TApiAuthorizationPolicyBeforeCreateContext<E>`
- `TApiAuthorizationPolicyBeforeGetContext<E>`
- `TApiAuthorizationPolicyBeforeGetListContext<E>`
- `TApiAuthorizationPolicyBeforeUpdateContext<E>`
- `TApiAuthorizationPolicyBeforePartialUpdateContext<E>`

### Permission matching

Permission helpers read granted permissions from `subject.permissions` and compare them to the required permission list passed into the helper:

- `match: EAuthorizationPermissionMatch.ANY` is the default
- `match: EAuthorizationPermissionMatch.ALL` requires every listed permission
- granted permissions may use wildcards such as `admin.user.*`, `admin.*`, or `*`
- required permissions should stay concrete, for example `admin.user.read`

### Custom subject resolvers

When `request.user` does not already match the default resolver heuristics, register a custom resolver with `ApiAuthorizationModule.forRoot({ subjectResolver })`.

The package also exports:

- `AuthorizationResolveDefaultSubject()` to reuse the built-in fallback
- `IApiAuthorizationSubjectResolver` for typing custom resolvers
- `AUTHORIZATION_SUBJECT_RESOLVER_TOKEN` for advanced Nest provider wiring

### Mixed role + permission example

```typescript
public onBeforeGetList(): TApiAuthorizationPolicyBeforeGetListResult<UserEntity> {
	return [
		...this.allowForRoles(["platform-admin"]),
		...this.allowForPermissions(["admin.user.read", "admin.user.list"], {
			match: EAuthorizationPermissionMatch.ALL,
			scope: ({ subject }) => ({
				where: { operatorId: subject.attributes?.operatorId },
			}),
		});
	];
}
```

### Hook context

`context.DATA` contains typed metadata for the request, including the resolved `subject`, optional `authenticationRequest`, and any request metadata captured by the guard.

### Return type helpers

Use `TApiAuthorizationPolicyBefore*Result` and `TApiAuthorizationPolicyCustomActionResult` to make hook return types explicit. Hooks always return arrays of rules; use an empty array (`[]`) when no rules apply.

### Policy cache override

Use the `cache` option when you want to cache only specific policies:

```typescript
@ApiAuthorizationPolicy<UserEntity>({
	entity: UserEntity,
	cache: { isEnabled: true, ttlMs: 60_000 },
})
export class UserAccessPolicy extends ApiAuthorizationPolicyBase<UserEntity> {}
```
